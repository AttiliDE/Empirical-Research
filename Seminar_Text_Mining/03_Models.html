1. Introduction

This notebook will be used to train and test models for sentiment analysis, more specifically, the goal is to choose the best model to apply on our MD&A-sections. As our own corpus is not labeled, we will be using a dataset similar to ours, namely the so-called financial phrasebank dataset, which will be addressed as finPB.

The dataset was created by Malo et. al (2013) and is avaiable in 3 different versions according to the level of aggreement of the annotators. We will use the 50% agreement threshold-dataset that can be downloaded from the Huggingface website.

The following model will be trained (when necessary) and their performance computed:

  • Dictionary-Based:
    • Loughran and McDonald (2011) implementation by pysentiment2. The dictionary will be addressed as LM.
  • Machine-Learning Based:
    • XGBoost
    • Support Vector Machine (SVM)
  • Transformer-based:
    • finBert

Notice that finBert is already trained and need not be trained again. It is very unlike that we could achieve an increase in performance than the one achieved by the authors, as these had much more data than available to us.

To the best of our knowledge there are many versions of finBert, one of them, and maybe the best known is the one trained by ProsusAI, neverthless, we will use the same dataset the authors used to train their model. In order not to bias the result in favor of finBert, we will use the finBert trained by Huang, Wang and Yang (2022) available for free use from HuggingFace.

In the dictionary case, there is no training at all, because the implementation of a dictionary is based on the net-count of positive and negative words and the pysentiment2 implementation automatically returns the level of polarity, so that we need to create a method to assign a classification to the phrases (negative, neutral or positive) based on this measure.

def parallelize_cleantext(df, processes = os.cpu_count(), text_column = 'raw_text'):
    """
    Function is used to parallelize the clean_text function (below)
    - Arguments:
      - df: dataframe with the text
      - processes: the number of processors used to parallelize the function
      - text_column: the name of the column containing the raw text
    - Returns: a copy of the original dataset with additional columns containing the cleaned texts
    """
    # copy the dataframe
    df = df.copy(deep = True)
    # get text column
    column_index = df.columns.get_loc(text_column)

    def clean_text(file_info, remove_specialchar = True, lower_text = True, lower_case = True, remove_sw = True):
        """
        Function will return the cleaned text
        """
        index = file_info[0]
        text = file_info[1]
        #print(index, text)

        if remove_specialchar:
            #pass
            text = re.sub(r'[^A-Za-z0-9\s]', ' ', text)
        if lower_text:
            text = text.lower()


        tokens = word_tokenize(text)

        if remove_sw:
            non_stopwords = [word for word in tokens if word.lower() not in nltk.corpus.stopwords.words('english')]
            text = ' '.join(non_stopwords)

        return index, text
    # given a dataframe with the texts we will yield the text rowwiese
    def yield_row(df):
        for row in df.itertuples():
            index = row.Index
            text = row._asdict()[text_column]

            yield index, text

    gen = yield_row(df)

    with ProcessPool(processes = processes) as pool:
        results = list(tqdm(
            pool.imap(clean_text, gen), total = len(df), unit = 'it', desc = 'Cleaning text'
        ))


    for result in results:
        index, cleaned_text = result
        df.at[index, 'cleaned_text'] = cleaned_text

    nlp = spacy.load("en_core_web_sm")
    counter = 0
    for row in tqdm(df.itertuples(), desc = 'Lemmatizing text', total = df.shape[0]):
        counter += 1
        doc = nlp(row._asdict()['cleaned_text'])
        lemmatized_text = ' '.join([token.lemma_ for token in doc])
        df.at[row.Index, 'cleaned_text'] = lemmatized_text
    return df

{'Positive': 0, 'Negative': 0, 'Polarity': 0.0, 'Subjectivity': 0.0}

Hence, we need now a scheme to classify the phrases. One way of doing this is by net-counting the words. We could either rely on polarity and assign values, e.g. between -0.5 and 0.5 to the neutral-class, values above 0.5 to the positive and below -0.5 to the negative or we could rely on the net-count of positive and negative words, labeling positive sentences as 1 and negative as -1 and summing up. So if the count is positive, we assign the phrase to the positive class and if it is negative, we assign to the negative class, otherwise it is neutral.

Due to the very nature of the polarity score, we know whether the net count is positive or negative by the score sign. Polarity is computed as:

\begin{align} polarity = \frac{N_{positive} - N_{negative}}{N} \end{align}

If polarity is greater (lower) than zero, then the number of positive (negative) words is greater than the number of negative (positive).

We will create a function with a lower and upper bound that allows us to predict the sentiment according to the intervals.

In [79]:
def assign_classLM(x, lb, ub):
  """
  Function takes a pandas dataframe containing a dictionary (as string) as input, transform it to literal and
  return whether the prediction is negative (0), neutral (1), or positive (2)
  Values above the upper bound are assigned to the class positive, values below are negative and values in-between are neutral.
  """
  eval_dict = ast.literal_eval(str(x))
  polarity = eval_dict['Polarity']

  if polarity < lb:
    return 0 # negative
  elif polarity > ub:
    return 2 # positive
  else:
    return 1 # neutral

# Method 1: simple net count (lb = up = 0)
y_pred_LM_v1 = y_LM.apply(assign_classLM, lb = 0, ub = 0).copy(deep = True)

# Method 2: negative if polarity < -0.5, positive if polarity > 0.5 and neutral otherwise
y_pred_LM_v2 = y_LM.apply(assign_classLM, lb = -0.5, ub = 0.5)

As seen in the output above, accuracy is very low, only 57.42% of the phrases are correctly predicted by this method.

Recall and precision and F1-score are also not much better. An easy definition of the formulas is provided by the website.

  • Precision:
    • \begin{align} \text{Precision}=\frac{TP}{TP+FP} \end{align}
  • Recall
    • \begin{align} \text{Recall}=\frac{TP}{TP+FN} \end{align}
  • F1-Score
    • \begin{align} \text{F1}_{\mathcal{S}}=2\frac{\text{Precision}\times\text{Recall}}{\text{Precision}+\text{Recall}} \end{align}

F1-score is a better measure of model performance, because it accounts for both, false positives and false negatives, which are missing in the recall and precision formulas respectively. Hence, we will be using the macro F1-score to compare the models and choose the model with the highest score.

We choose macro-average because of the inbalance of the dataset. Another alternative would be to use the weighted average, but this would put more emphasis on the neutral class, as the class is over represented; for our case all classes are equally important.

2.2.1 Model Training

2.2.1.1. XGBoost

The first model that we will train and compute performance is XGBoost. XGBoost is a so-called "ensamble model". It uses sevearal "weak learnings" to build a powerful predictive model, in our case, decision trees. It starts with an intial prediction (usually the average) and build sucessively trees that aim to minimize the errors of past predictions. Further, the model allows for regularization parameters to encourage prouning of the trees and allows for specification of the max-depth of each tree.

A more detailed explaination of the model is provided by the official paper.

XGBoost: Model Training I (Using Raw-Data)

Just as we did with the dictionary case, we will split the data into test and train and apply 5-fold cross validation in the train set to find the best parameters and vectorization method.

We will first apply the model to the raw phrases, meaning that we will only tokenize the data.

In a second try we will use the cleaned phrases as done in Section 1 (introduction) and check whether we can achieve higher performance.

# We create a function similar to the older one to create CM
def plot_cm_multiaxes(y_test, Y_pred, labels = None, ann_size = 12, cbar_fontsize = 12, x_ticklabel_size = 12, y_ticklabel_size = 12, **kwargs):

  """
  Function plots confusion matrices in a single canvas.
  Arguments:
    - y_test: pandas series (or numpy vector) containing the true labels.
    - Y_pred: a pandas dataframe containing the predictions
    - Labels: the labels to be used in the plot.
    - ann_size: the size of the annotation-font.
    - cbar_fontsize: the size of the font used in the bar in the right side of the graph.
    - x_ticklabel_size: size of the font used to label the x-axis
    - y_ticklabel_size: size of the font used to label the y-axis
    - kwargs:
      - titles: a list of titles to pass to the plots. If not passed, the titles will be "Confunion Matrix:" followed by the names of the columns in the Y_pred dataset.
      - figsize: a tuple with the dimensions of the canvas for the plot
      - sharex: a boolean specifying whether the X axis should be shared
      - sharey: a boolean specifying whether the y axis should be shared
  """

  # get the number of columns in the Y_pred dataset, it will be the same number of axes we need
  axes_num = Y_pred.shape[1]

  default_titles = ['Confusion Matrix: ' + str(column) for column in Y_pred.columns.tolist()]
  titles = kwargs.get('titles', default_titles)
  default_figsize = (4, 4)
  figsize = kwargs.get('figsize', default_figsize)
  sharex = kwargs.get('sharex', True)
  sharey = kwargs.get('sharey', True)
  # make sure that figsize was passed as a tuple
  assert isinstance(figsize, tuple), 'figsize must be passed as a tuple containing the width and the height: (width, height)'
  #get unique labels
  num_labels = unique_labels(y_test)

  if labels != None:
      assert isinstance(labels, dict), 'Labels need to be passed as a dictionary mapping numbers to representations'
      try:

          labels = [labels[i] for i in num_labels]
      except:
          labels = [labels[str(i)] for i in num_labels]
  else:
      labels = num_labels

  # create a dictionary to store the confusion matrices
  confusion_matrices = {}
  # now iterate over the columns of the dataset with the predictions and compute the confusion matrices
  for column in Y_pred.columns.tolist():

    confusion_matrices[column] = metrics.confusion_matrix(y_test, Y_pred[column].tolist(), labels = num_labels)

  # in the figure, a row will have two axes, so the number of rows we need will be the number of axes divided by 2 (rounded up in the case of uneven number)
  n_rows = math.ceil(axes_num / 2)
  n_cols = 2 if n_rows > 1 else 1

  # create the subplots
  fig, axes = plt.subplots(n_rows, n_cols, figsize = figsize, sharex = sharex, sharey = sharey)

  # Now we check whether we fill in all axes or if there is an axis without data. In this case, we make this axis (the last one) invisible
  if axes_num % 2 > 0 and n_rows > 1:
    axes[-1, -1].set_visible(False)

  # we now iterate over the axis and plot the results
  counter = 0
  for i, ax_row in enumerate(axes):

    for j, ax in enumerate(ax_row):

      # we create an exception, because if we have more axis than data, it will cause an error when we try to plot the last graph
      try:

        cm_name = list(confusion_matrices)[counter]

        heatmap = sns.heatmap(
            confusion_matrices[cm_name],
            annot = True,
            fmt = 'd',
            cmap = 'Blues',
            xticklabels = labels,
            yticklabels = labels,
            ax = ax,
            annot_kws = {'size': ann_size}
        )
        cbar = heatmap.collections[0].colorbar
        cbar.ax.tick_params(labelsize=cbar_fontsize)

        if j == 0:
          ax.set_ylabel('True label', labelpad = 20, fontsize = y_ticklabel_size)
        if i == (len(axes) - 1):
          ax.set_xlabel('Predicted', labelpad = 20, fontsize = x_ticklabel_size)
        ax.set_title(titles[counter], pad = 5, weight = 'bold')
        counter += 1
      except:
        pass
  return fig, axes
